What is an Array?

An array is a linear data structure that collects elements of the same data type and stores them in contiguous memory locations. Think of it as a row of boxes, where each box can hold one item, and you can access any box by knowing its position number (index).


# Creating an array
arr = [1, 2, 3, 4, 5]
print(f"Original array: {arr}")

Key Characteristics

üì¶ Fixed Size

Arrays have a predetermined size that cannot be changed during runtime

üî¢ Indexed Access

Elements can be accessed directly using their index position (0-based)

üíæ Contiguous Memory

Elements are stored in consecutive memory locations

üéØ Same Data Type

All elements must be of the same data type

üïê Time Complexity Analysis (Big O Notation)

ü§î What is Time Complexity?

Time complexity is like measuring how fast you can find a book in different types of libraries. It tells us how the time to complete a task grows as the amount of data increases.

Think of it as asking: "If I have 10 books vs 1000 books, how much longer will it take to find what I'm looking for?"

üìö Real-World Examples

O(1) Constant Time

Like: Opening a specific page in a book when you know the page number

Array Example: Getting arr[5] - you directly jump to position 5

element = arr[index] // Always takes same time

O(n) Linear Time

Like: Reading every page to find a specific word

Array Example: Searching for a value - you might need to check every element

for i in range(len(arr)): // Time grows with array size

O(log n) Logarithmic Time

Like: Finding a word in a dictionary by splitting it in half repeatedly

Array Example: Binary search in a sorted array

binary_search(sorted_arr, target) // Very efficient!

O(n¬≤) Quadratic Time

Like: Comparing every person with every other person in a room

Array Example: Bubble sort - nested loops

for i in arr: for j in arr: // Gets slow quickly!

üèÉ‚Äç‚ôÇÔ∏è Array Operations & Their Time Complexity

üéØ Access by Index: arr[i]

O(1)

Why O(1)? Arrays store elements in consecutive memory locations. The computer can calculate the exact address instantly using: base_address + (index √ó element_size)

arr[0] takes same time as arr[1000] - both are instant!

üîç Search for Value: find(x)

O(n)

Why O(n)? In worst case, we might need to check every element. If the value is at the end or doesn't exist, we check all n elements.

Searching for 5 in [1,2,3,4,5] - might check all 5 elements

‚ûï Insert/Delete: add/remove at position

O(n)

Why O(n)? When inserting/deleting in middle, we need to shift all elements after that position. More elements = more shifting required.

Insert 'X' at index 1: [A,B,C,D] ‚Üí [A,X,B,C,D] (shift B,C,D)

üí° Key Takeaways

  • ‚úì Arrays are great for: Fast access by index, when you know the position
  • ‚ö†Ô∏è Arrays are slower for: Searching unknown values, frequent insertions/deletions
  • üéØ Remember: Time complexity helps us choose the right data structure for our needs

Array Implementation Examples
Python
def array_operations():
    # Creating an array
    arr = [1, 2, 3, 4, 5]
    print(f"Original array: {arr}")

    # Accessing elements
    first_element = arr[0]  # Output: 1
    last_element = arr[-1]  # Output: 5
    print(f"First: {first_element}, Last: {last_element}")

    # Modifying elements
    arr[2] = 10  # arr becomes [1, 2, 10, 4, 5]
    print(f"After modification: {arr}")

    # Array length
    length = len(arr)  # Output: 5
    print(f"Array length: {length}")
    
    # Adding elements
    arr.append(6)  # [1, 2, 10, 4, 5, 6]
    arr.insert(1, 15)  # [1, 15, 2, 10, 4, 5, 6]
    print(f"After insertions: {arr}")
    
    # Removing elements
    arr.pop()  # removes last element
    arr.remove(15)  # removes first occurrence of 15
    print(f"After removals: {arr}")
    
    # Searching
    if 10 in arr:
        index = arr.index(10)
        print(f"Element 10 found at index: {index}")
    
    # Slicing
    sub_array = arr[1:4]  # elements from index 1 to 3
    print(f"Sliced array: {sub_array}")

# Call the function
array_operations()

Types of Arrays

üî¢ Static Arrays

Fixed size arrays where the size is determined at compile time.

int arr[5] = {1, 2, 3, 4, 5};

üìà Dynamic Arrays

Resizable arrays that can grow or shrink during runtime.

vector<int> arr = {1, 2, 3};

üíª Practice Problems

1. Two Sum

Easy

Given an array of integers and a target sum, return indices of two numbers that add up to the target.

Example: nums = [2,7,11,15], target = 9 ‚Üí Output: [0,1]

2. Maximum Subarray

Medium

Find the contiguous subarray with the largest sum using Kadane's algorithm.

Example: nums = [-2,1,-3,4,-1,2,1,-5,4] ‚Üí Output: 6

3. Merge Intervals

Hard

Given a collection of intervals, merge all overlapping intervals and return the result.

Example: intervals = [[1,3],[2,6],[8,10],[15,18]] ‚Üí Output: [[1,6],[8,10],[15,18]]
